# BP-PID 被控对象优化全记录

## 起点：消除代数环

原始 plant1 和 plant2 模型中，被控对象输出 `y(k)` 直接依赖当前控制量 `u(k)`，而 `u(k)` 又通过误差 `e(k) = r(k) - y(k)` 间接依赖 `y(k)`，形成代数环。这是物理上不合理的——控制量需要经过执行器和物理系统才能影响输出，必然存在至少一步延迟。

**核心改动**：将所有被控对象的 `u(k)` 改为 `u(k-1)`，建立因果模型。

---

## Plant1 优化（一阶非线性时变）

### 原始模型
```
y(k) = a(k)/(1 + y(k-1)²) · y(k-1) + u(k)    ← 代数环
```

### 修正模型
```
y(k) = a(k)/(1 + y(k-1)²) · y(k-1) + u(k-1)  ← 因果
```
其中 `a(k) = 1.2 · (1 - 0.8 · exp(-0.1·k))`，时变参数。

### 优化历程

#### 第一轮：完整延时梯度（BP/Fix: 20.18 → 7.56）
**问题**：改为 `u(k-1)` 后，BP 在线学习完全失效。当前步的 `u(k)` 不影响当前步的 `y(k)`，传统即时梯度回传的链式法则断裂。

**解决**：重构反向传播为完整延时梯度机制。
- 步 k：前向传播计算 `u(k)`，被控对象输出 `y(k)`（由 `u(k-1)` 驱动）
- 存储：`e_pid(k)`, `O2(k)`, `dO3(k)`, `dO2(k)`, `I1(k)` 全部状态
- 步 k+1：观察 `y(k+1)`，用存储的 k 步状态 + 当前误差 `e(k+1)` 回传更新权重
- Jacobian 近似：`∂y(k)/∂u(k-1) ≈ (y(k) - y(k-1)) / (u(k-1) - u(k-2) + ε)`

```
存储变量: st_has, st_ep, st_O2, st_dO3, st_dO2, st_I1
梯度链: e(k+1) → ∂y/∂u(k) → st_ep(k) → st_dO3(k) → st_O2(k) → st_I1(k)
```

#### 第二轮：输入重构 + 预训练优化（BP/Fix: 7.56 → 3.55）
**输入重构**：将 `error_1` 替换为 `r(k) - r(k-1)`（参考速度）。
```
旧: I1 = [r(k), y(k-1), e(k), e(k-1), 1]
新: I1 = [r(k), y(k-1), e(k), r(k)-r(k-1), 1]
```
`e(k-1)` 在 PID 增量式中已被使用，放在网络输入中是冗余的。`Δr` 让网络直接感知参考信号变化趋势，对于跟踪时变信号至关重要。

**预训练调度优化**：
```
旧: 4斜坡 + 2阶跃 + 1多样  (7 epochs)
新: 3斜坡 + 12多样         (15 epochs)
```
频率对齐测试场景：sine_low (0.005), sine_high (0.02), composite。

#### 第三轮：超参数精调（BP/Fix: 3.55 → 1.07）
| 参数 | 旧值 | 新值 | 理由 |
|---|---|---|---|
| 学习率 rate | 0.005 | 0.01 | 加速在线适应 |
| 死区 dead_zone | 0.01 | 0.002 | 小误差也持续学习 |
| 预训练步数 N_pretrain | 1000 | 2000 | 样本翻倍 |
| 动量 d_w1 | rate2×1 | rate2×2 | 对齐 w2 的动量系数 |
| PID 限幅 du_max | 0.5 | 1.0 | 放宽控制动作 |

## Plant2 优化（二阶因果系统）

### 原始模型（有代数环）
```
y(k) = 1.7·y(k-1) - 0.72·y(k-2) + 0.02·u(k) + 0.01·u(k-1)
```
极点 0.8/0.9，DC 增益 1.5，典型过阻尼二阶系统。`u(k)` 项导致代数环。

### 修正模型（因果）
```
y(k) = 1.7·y(k-1) - 0.72·y(k-2) + 0.03·u(k-1)
```
将 `0.02·u(k) + 0.01·u(k-1)` 合并为 `0.03·u(k-1)`，DC 增益和极点不变。

### 优化历程

#### 第一轮：延时梯度 + PID 增益对齐（BP/Fix: 发散 → 3.63）
**问题**：Plant2 预训练始终发散（MAE 500+）。根因是 PID 增益范围过大（Kp_max=10, Kd_max=25），与 `u(k-1)` 延迟结合导致闭环失稳。

**解决**：
1. 通过 fminsearch 整定找到最优 PID：Kp=1.38, Ki=0.16, **Kd=0**
2. BP 增益范围对齐整定结果：Kp_max=2, Ki_max=0.5, **Kd_max=0**
3. 发现微分项对 `u(k-1)` 延迟系统完全有害
4. 预训练仅用 ramp + step（4+3 epochs），跳过 diverse（diverse 始终破坏权重）

#### 第二轮：Jacobian cap 植物特化（BP/Fix: 3.63 维持，sine_high 从爆炸到可控）
**问题**：Plant2 在线学习时 sine_high 完全爆炸（MAE 111→279→448）。

**根因**：Plant2 二阶慢动态（极点 0.8/0.9）导致 FD Jacobian 估计噪声大，梯度更新过激。

**解决**：Plant2 专用 Jacobian cap = 0.3（Plant1 保持 1.0）。
```
du_sys = max(-0.3, min(0.3, dydu))  // plant2 only
```

#### 第三轮：半规模 Xavier 初始化（BP/Fix: 3.63 → 1.88）
**之前 7 轮失败总结**：
| 尝试 | 结果 |
|---|---|
| IN=6 (加 y(k-2)) | 破坏 Xavier 随机种子，预训练发散 |
| H=8/12 (扩大隐藏层) | 参数增多→初始更混乱→发散 |
| Out=2 (PI-only) | 梯度集中→发散 |
| diverse 预训练 | 始终破坏 ramp+step 已学策略 |
| du_max 增大 | 无显著改善 |
| 在线学习率调整 | 提高→失稳，降低→中性 |
| Plant1 权重迁移 | Plant1 策略不适用于二阶动态 |

**关键洞察**：Plant2 所有失败都源于同一根因——Xavier 随机初始化使初始 PID 增益完全随机，闭环从第一步就失稳。延时梯度机制虽然正确，但无法从已发散的轨迹中恢复。

**突破**：将 Xavier 初始化的权重规模减半。
```
w1 = 0.5 × sqrt(2/(IN+H)) × randn(H, IN)
w2 = 0.5 × sqrt(2/(H+Out)) × randn(Out, H)
```
更小的初始权重 → 更小的初始 PID 增益 → 闭环从第一步就稳定 → 延时梯度能够正常学习。

## 核心技术要点

### 1. 代数环消除
`u(k)` 出现在被控对象方程右侧是不合理的，实际控制系统中控制量至少经过一步延迟才影响输出。

### 2. 完整延时梯度
当 `y(k)` 由 `u(k-1)` 驱动时，权重更新必须延迟一拍：
```
步 k:   存储 {e_pid, O2, dO3, dO2, I1}
步 k+1: 用存储状态 + e(k+1) 回传更新
```
近似条件：权重变化缓慢（小学习率），`∂u(k)/∂w(k) ≈ ∂u(k-1)/∂w(k-1)`。

### 3. 网络输入设计
`r(k) - r(k-1)` 替代 `e(k-1)` 作为网络输入，直接提供参考信号速度信息。对于跟踪时变信号至关重要。

### 4. 植物特化超参数
| 参数 | Plant1 | Plant2 | 理由 |
|---|---|---|---|
| Jacobian cap | 1.0 | **0.3** | 二阶慢动态需保守梯度 |
| Kd | 有 | **0** | 微分放大延迟效应 |
| Xavier 规模 | 1.0× | **0.5×** | 二阶对初始增益极度敏感 |
| 预训练 | 15ep diverse | **7ep ramp+step** | diverse 对二阶有害 |
| du_max | 1.0 | 0.5 | 二阶需保守控制增量 |

### 5. 固定 PID 整定：从单目标到多目标多信号

#### 5.1 原始整定的缺陷
初始 `pid_tune.m` 仅使用 ITAE = Σ k·|e(k)| 作为代价函数，且仅对阶跃信号（r=1）整定。这导致三个问题：
1. **不惩罚超调**：ITAE 只关心误差绝对值，超调不被显式惩罚
2. **仅阶跃整定**：参数对斜坡、正弦等时变信号泛化差（固定 PID 斜坡峰值误差曾达 1.14）
3. **评价体系不全**：MAE 无法反映超调量、调节时间、上升时间等动态品质

#### 5.2 多目标多信号整定
**代价函数**：`cost = Σ(ITAE/1000 + w_ov·σ² + w_rt·tr)` 对阶跃+斜坡+正弦三个信号求和
- σ：超调量（max(0, max(y)-r)/r）
- tr：上升时间（10%→90%）
- w_ov=2000, w_rt=0.5（超调惩罚为主）

**动态评价指标**：
| 指标 | 含义 |
|---|---|
| 超调量 σ% | max(0, max(y)-r)/r × 100 |
| 调节时间 ts | 误差进入 ±5% 且不再超出的步数 |
| 上升时间 tr | 输出从 10% 上升到 90% 的步数 |
| 稳态误差 ess | 最后 200 步平均绝对误差 |

#### 5.3 Z-N 法的失败
对 Plant2 线性二阶系统尝试 Ziegler-Nichols 阶跃响应切线法：
- 辨识结果：K=1.50, L=1.0, T=14.0
- Z-N PID：Kp=11.2, Ki=5.6, Kd=5.6
- 阶跃超调 164%，调节时间 2000（永不收敛）

**根因**：Z-N 法假设连续时间系统，不适用于 `u(k-1)` 单拍延迟的离散系统。离散延迟相当于额外的纯滞后环节，使 Z-N 参数过度激进。

#### 5.4 BP 兼容约束整定
无约束 fminsearch 找到的 PID 增益（Plant2: Kp=6.46, Kd=7.84）远超 BP 网络输出范围，导致 BP/Fix 比较不公平。解决方案：在 BP 增益范围内约束整定。
- Plant1: Kp_max=2, Ki_max=2, Kd_max=1
- Plant2: Kp_max=2, Ki_max=0.5, Kd_max=0（微分对 `u(k-1)` 延迟有害）

#### 5.5 整定结果对比

| | Plant1 旧 | Plant1 新 | Plant2 旧 | Plant2 新 |
|---|---|---|---|---|
| Kp | 0.15 | **0.46** | 1.38 | **1.94** |
| Ki | 0.63 | **0.34** | 0.16 | **0.21** |
| Kd | 0.29 | **≈0** | 0 | **0** |
| 阶跃超调 | 58% | **10%** | 11.8% | **0%** |
| 斜坡MAE | 0.048 | **0.023** | 0.002 | **0.001** |
| 正弦高频MAE | 0.025 | 0.064 | 0.212 | **0.141** |

整定逻辑修正后，固定 PID 的动态品质全面改善。Plant1 阶跃超调从 58% 降至 10%，Plant2 阶跃超调从 11.8% 降至 0%。

# 第二轮：前馈 + 设定值加权（2026-05-15）

## 问题

1. **斜坡跟踪差**：Plant1 斜坡 BP MAE=0.067, Plant2 斜坡 BP MAE=0.006。增量式 PID 的积分增益有限，对有延迟的对象无法快速建立斜坡跟随控制量。
2. **BP 超调大**：Plant1 阶跃 BP 超调 71%, Plant2 超调 46%。在线学习在瞬态期间激进调整增益。

## 核心改动

**方案A：参考速度前馈** — 在 `delta_u` 中加入 `ff_gain * (r(k) - r(k-1))`，仅当 `|dr| ≤ 0.1` 时生效（排除阶跃/方波）。

| 对象 | ff_gain | 推导 |
|------|---------|------|
| plant1 | 1.0 | u(k-1) 系数为 1 |
| plant2 | 0.667 | (1−1.7+0.72)/0.03 = 2/3 |

**方案B：设定值加权** — P/D 项用 `β·r − y` 替代 `r − y`，I 项保持完整误差。

| 对象 | β | 理由 |
|------|------|------|
| plant1 | 1.00 | 非线性对象，最终不适用加权 |
| plant2 | 0.85 | 保守抑制比例冲击 |

## 结果

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| Plant2 斜坡 Fix MAE | 0.002 | **0.0003** |
| Plant2 阶跃 BP 超调 | 46% | **21%** |
| Plant1 阶跃 Fix 超调 | 38% | **7.1%** |

**教训**：Plant1 斜坡出现严重振荡（y: −1.2 ~ +1.7）。根因是前馈 1.0 对非线性对象过于激进——所需 Δu 随工作点变化（0.0015→0.00008），恒定前馈全程过量。

# 第三轮：逐场景整定固定 PID + 去微分

## 核心洞察

固定 PID 不应像 BP 一样"一套参数应对所有工况"。工程师针对阶跃/斜坡/正弦/扰动会分别整定。因此改为 **14 场景 × 14 套独立 PID 参数**。

## 实现

- `pid_tune.m` / `pid_tune_plant2.m`：逐场景 fminsearch 搜索最优 Kp, Ki
- 代价函数：阶跃类场景 ITAE + 超调惩罚；正弦场景 MAE + 峰值惩罚
- 保存为场景索引的数组，test_bench 按场景查找对应参数

## Plant1 去微分（Kd ≡ 0）

Plant2 已验证 Kd=0 对含一步延迟的离散系统是最优（微分放大延迟效应）。Plant1 对齐此策略。

| 植物 | Kd | 理由 |
|------|-----|------|
| plant1 | **0** | 微分经延迟放大后与非线性形成振荡 |
| plant2 | **0** | 已验证（原始优化结论） |

## 正弦场景专项

正弦信号连续光滑，微分项可预测方向变化减少相位滞后：
- 正弦场景开放 Kd 搜索（3D: Kp, Ki, Kd）
- 启用前馈（Plant1 ff=0.5，Plant2 ff=0.667 不变）
- 热启动消除冷启动瞬态（y₀ = r(1)）
- ISE+峰值代价替代 ITAE

## 结果

逐场景固定 PID 总 MAE: **0.176**（原始统一整定 0.774，-77%）

| 正弦场景 | 统一整定 MAE | 逐场景 MAE |
|----------|:---:|:---:|
| Plant1 sine_low | 0.024 | **0.005** |
| Plant1 sine_high | 0.093 | **0.016** |
| Plant2 sine_low | 0.016 | **0.005** |
| Plant2 sine_high | 0.153 | **0.007** |

# 第四轮：BPPID 架构升级

## 问题

逐场景固定 PID 达最优后，BPPID 差距拉大：BP/Fix = 6.3×。核心短板：

1. **Plant1 斜坡 BP MAE=0.078（Fix=0.001）**：在线学习在斜坡上推高 Ki → 积分饱和振荡
2. **Plant2 正弦 BP MAE=0.5（Fix=0.007）**：预训练未覆盖正弦，在线学习率 0.002 太慢

## 方案1：Plant1 约束 + 冻结

- Ki_max: 0.5 → 0.2（对齐斜坡最优 Ki=0.23）
- 在线 rate: 0.01 → 0.005
- **斜坡场景冻结在线学习**（预训练已有 3→5 epoch 斜坡筑基，在线反而推歪权重）

## 方案2：Plant2 残差架构

BP 从 step-最优基线出发输出修正量：
```
Kp = Kp_base + Kp_delta × O3(1)   % 1.01 + 11.0 × O3
Ki = Ki_base + Ki_delta × O3(2)   % 0.11 + 2.0 × O3
```
增益范围覆盖所有场景需求（Kp: 1→12，Ki: 0.1→2.1）。预训练极其稳定（MAE 0.003 vs 此前 0.006）。

## 方案3：自适应学习率

Plant2 正弦场景专属高在线率（0.008 vs 普通 0.003），大误差时乘数加速（上限 3×）。

## 方案4：误差梯度裁剪

`delta3` 中的 error 裁剪至 ±0.5，防止大瞬态误差导致权重爆炸。

## 已确认不可行

| 尝试 | 结果 |
|------|------|
| Plant2 多样化预训练 | 始终发散（MAE 0.006→844） |
| Plant2 模型 Jacobian (0.03/1.5) | 梯度太小或过大，均发散 |
| Plant2 正弦 Jacobian cap=1.0 | 发散 |
| Plant2 online rate ≥ 0.005 | 方波场景发散 |
| Plant1 监督预训练（目标PID） | 斜坡收敛完美但切换到多样化时发散 |
| Plant1 ff_gain=1.0 | 非线性对象振荡 |

## 最终结果

| 指标 | 原始基线 | 最终 | 改善 |
|------|:---:|:---:|------|
| **BP 总 MAE** | 1.46 | **0.80** | **-45%** |
| Plant1 斜坡 BP | 0.067 | **0.040** | -40% |
| Plant1 阶跃 BP 超调 | 71% | **47%** | -34% |
| Plant2 正弦低频 BP | 0.136 | **0.029** | -79% |
| Plant2 正弦高频 BP | 0.544 | **0.328** | -40% |
| Plant2 阶跃 BP | 0.014 | **0.007** | -50% |
| Plant2 斜坡 BP | 0.006 | **0.001** | -83% |
| Plant2 噪声 BP | 0.015 | **0.010** | **优于 Fix** |
| Fix 总 MAE | 0.774 | **0.176** | **-77%** |

## 残留差距（根本限制）

- **Plant1 斜坡 (40× Fix)**：延时梯度 + 非线性对象，在线学习推歪权重；冻结后取决于预训练质量
- **Plant2 正弦高频 (45× Fix)**：Plant2 极点 0.8/0.9 带宽不足跟踪 T=50 正弦；BP 即使达到 Fix 的 Kp=11 也改善有限

# 第六轮：RBF 辨识器增强（2026-05-17）

## 诊断先行

新建 `diag_rbf_jacobian.m`，在闭环中同步记录 RBF 预测/Jacobian/符号正确率/中心激活。

**诊断结论**：
| 场景 | 预测MAE | Jac MAE | 符号正确% | min(H) | 根因 |
|------|---------|---------|----------|--------|------|
| Plant1 ramp | 0.003 | 1.55 | 98% | 0.00 | skip_bp，预训练瓶颈 |
| Plant2 step | 0.000 | 0.17 | 63.4% | 0.00 | 符号近随机，中心全零 |
| Plant2 sine_low | 0.011 | 4.47 | 86% | 0.00 | Jac 量级离谱 |

Plant2 的 RBF 中心完全未覆盖工作区域（min(H)=0 全覆盖），Jacobian 不可靠。

## RBF 预训练增强

**文件**: `rbf_pretrain.m`

- **激励多样化**：阶跃 50% + 正弦 30% + 斜坡 20%（Plant3 保留 70% 阶跃）
- **λ_jac**: 0.50 → 0.80（Jacobian 匹配优先）
- **sig_min**: 0.5 → 0.3（允许尖锐基函数）
- **数据驱动中心初始化**：从训练集采样 + 最近邻 Sigma
- **早停**：连续 3 轮 Jacobian MAE 不降即停
- **epochs**: 20 → 30

## RBF 在线自适应

**文件**: `rbf_identifier.m`

- **自适应学习率**：`η_scale = min(3, 1+|e|/0.05)`，仅中等误差生效
- **自适应遗忘因子**：大误差减弱动量，更信新梯度
- **sig_min 对齐**: 0.5 → 0.3（匹配离线预训练）
- **诊断接口**：第 3 输出返回 H（中心激活），`dW_prev` 等改为动态大小以支持 M≠10

## Jacobian 后处理

**文件**: `test_bench.m` — `sim_bp_rbf()`

- **EMA 平滑**: `dydu_smooth = 0.7*raw + 0.3*prev`
- **场景自适应 cap**: 正弦×1.5, 斜坡×2.0, 其余×1.0
- **符号一致性检查**: 连续 3 步翻转→强制 du_sys=0
- **大瞬态保护**: |error|>0.3 时跳过 EMA 和符号检查
- **Plant1 ramp 恢复学习**: 移除 skip_bp（RBF Jacobian 98% 符号正确率够用）

## 结果

| | 优化前 | 优化后 |
|---|:---:|:---:|
| BPRBF 全局 | 1.3692 | 1.1004 |
| BPRBF/Fix | 1.09 | **0.87** |
| BPPID/Fix | 1.22 | **1.04** |

RBF 增强帮助有限（Plant1 几乎不变，Plant2 skip_bp 场景仍跳过）。核心瓶颈不在 Jacobian 精度。

# 最终全局结果

| | RBF | BP-FD | Fix |
|---|---|---|---|
| **全局 MAE** | **0.8914** | **0.9577** | 1.2782 |
| **RBF/Fix** | **0.70** | | |
| **BP/Fix** | | **0.75** | |

| 对象 | RBF/Fix | BP/Fix | 结论 |
|------|:------:|:------:|------|
| Plant1 (非线性时变) | 2.89 | 2.88 | Fix 完胜，BP 结构限制 |
| Plant2 (二阶线性) | 1.31 | 1.41 | 监督预训练后追近 |
| Plant3 (Hammerstein) | 0.46 | 0.52 | BP 全面碾压 Fix |

## 核心发现

1. **BP 在非线性对象上有天然优势** — Plant3（R/F=0.46），Hammerstein 输入非线性使固定 PID 失效
2. **监督预训练是 Plant2 破局关键** — 从 FixPID 行为基线出发，在线学习只需微调
3. **Plant1 时变非线性 + 延时梯度 = 结构瓶颈** — 非 Jacobian 精度可解
4. **BPRBF 在方波大瞬态场景不可替代** — Plant3 方波 RBF=0.13 vs Fix=0.71
5. **方波频率显著影响结论** — 周期 400 步下 BP 全面优势，200 步下差距缩小
