# BP-PID 被控对象优化全记录

## 起点：消除代数环

原始 plant1 和 plant2 模型中，被控对象输出 `y(k)` 直接依赖当前控制量 `u(k)`，而 `u(k)` 又通过误差 `e(k) = r(k) - y(k)` 间接依赖 `y(k)`，形成代数环。这是物理上不合理的——控制量需要经过执行器和物理系统才能影响输出，必然存在至少一步延迟。

**核心改动**：将所有被控对象的 `u(k)` 改为 `u(k-1)`，建立因果模型。

---

## Plant1 优化（一阶非线性时变）

### 原始模型
```
y(k) = a(k)/(1 + y(k-1)²) · y(k-1) + u(k)    ← 代数环
```

### 修正模型
```
y(k) = a(k)/(1 + y(k-1)²) · y(k-1) + u(k-1)  ← 因果
```
其中 `a(k) = 1.2 · (1 - 0.8 · exp(-0.1·k))`，时变参数。

### 优化历程

#### 第一轮：完整延时梯度（BP/Fix: 20.18 → 7.56）
**问题**：改为 `u(k-1)` 后，BP 在线学习完全失效。当前步的 `u(k)` 不影响当前步的 `y(k)`，传统即时梯度回传的链式法则断裂。

**解决**：重构反向传播为完整延时梯度机制。
- 步 k：前向传播计算 `u(k)`，被控对象输出 `y(k)`（由 `u(k-1)` 驱动）
- 存储：`e_pid(k)`, `O2(k)`, `dO3(k)`, `dO2(k)`, `I1(k)` 全部状态
- 步 k+1：观察 `y(k+1)`，用存储的 k 步状态 + 当前误差 `e(k+1)` 回传更新权重
- Jacobian 近似：`∂y(k)/∂u(k-1) ≈ (y(k) - y(k-1)) / (u(k-1) - u(k-2) + ε)`

```
存储变量: st_has, st_ep, st_O2, st_dO3, st_dO2, st_I1
梯度链: e(k+1) → ∂y/∂u(k) → st_ep(k) → st_dO3(k) → st_O2(k) → st_I1(k)
```

#### 第二轮：输入重构 + 预训练优化（BP/Fix: 7.56 → 3.55）
**输入重构**：将 `error_1` 替换为 `r(k) - r(k-1)`（参考速度）。
```
旧: I1 = [r(k), y(k-1), e(k), e(k-1), 1]
新: I1 = [r(k), y(k-1), e(k), r(k)-r(k-1), 1]
```
`e(k-1)` 在 PID 增量式中已被使用，放在网络输入中是冗余的。`Δr` 让网络直接感知参考信号变化趋势，对于跟踪时变信号至关重要。

**预训练调度优化**：
```
旧: 4斜坡 + 2阶跃 + 1多样  (7 epochs)
新: 3斜坡 + 12多样         (15 epochs)
```
频率对齐测试场景：sine_low (0.005), sine_high (0.02), composite。

#### 第三轮：超参数精调（BP/Fix: 3.55 → 1.07）
| 参数 | 旧值 | 新值 | 理由 |
|---|---|---|---|
| 学习率 rate | 0.005 | 0.01 | 加速在线适应 |
| 死区 dead_zone | 0.01 | 0.002 | 小误差也持续学习 |
| 预训练步数 N_pretrain | 1000 | 2000 | 样本翻倍 |
| 动量 d_w1 | rate2×1 | rate2×2 | 对齐 w2 的动量系数 |
| PID 限幅 du_max | 0.5 | 1.0 | 放宽控制动作 |

## Plant2 优化（二阶因果系统）

### 原始模型（有代数环）
```
y(k) = 1.7·y(k-1) - 0.72·y(k-2) + 0.02·u(k) + 0.01·u(k-1)
```
极点 0.8/0.9，DC 增益 1.5，典型过阻尼二阶系统。`u(k)` 项导致代数环。

### 修正模型（因果）
```
y(k) = 1.7·y(k-1) - 0.72·y(k-2) + 0.03·u(k-1)
```
将 `0.02·u(k) + 0.01·u(k-1)` 合并为 `0.03·u(k-1)`，DC 增益和极点不变。

### 优化历程

#### 第一轮：延时梯度 + PID 增益对齐（BP/Fix: 发散 → 3.63）
**问题**：Plant2 预训练始终发散（MAE 500+）。根因是 PID 增益范围过大（Kp_max=10, Kd_max=25），与 `u(k-1)` 延迟结合导致闭环失稳。

**解决**：
1. 通过 fminsearch 整定找到最优 PID：Kp=1.38, Ki=0.16, **Kd=0**
2. BP 增益范围对齐整定结果：Kp_max=2, Ki_max=0.5, **Kd_max=0**
3. 发现微分项对 `u(k-1)` 延迟系统完全有害
4. 预训练仅用 ramp + step（4+3 epochs），跳过 diverse（diverse 始终破坏权重）

#### 第二轮：Jacobian cap 植物特化（BP/Fix: 3.63 维持，sine_high 从爆炸到可控）
**问题**：Plant2 在线学习时 sine_high 完全爆炸（MAE 111→279→448）。

**根因**：Plant2 二阶慢动态（极点 0.8/0.9）导致 FD Jacobian 估计噪声大，梯度更新过激。

**解决**：Plant2 专用 Jacobian cap = 0.3（Plant1 保持 1.0）。
```
du_sys = max(-0.3, min(0.3, dydu))  // plant2 only
```

#### 第三轮：半规模 Xavier 初始化（BP/Fix: 3.63 → 1.88）
**之前 7 轮失败总结**：
| 尝试 | 结果 |
|---|---|
| IN=6 (加 y(k-2)) | 破坏 Xavier 随机种子，预训练发散 |
| H=8/12 (扩大隐藏层) | 参数增多→初始更混乱→发散 |
| Out=2 (PI-only) | 梯度集中→发散 |
| diverse 预训练 | 始终破坏 ramp+step 已学策略 |
| du_max 增大 | 无显著改善 |
| 在线学习率调整 | 提高→失稳，降低→中性 |
| Plant1 权重迁移 | Plant1 策略不适用于二阶动态 |

**关键洞察**：Plant2 所有失败都源于同一根因——Xavier 随机初始化使初始 PID 增益完全随机，闭环从第一步就失稳。延时梯度机制虽然正确，但无法从已发散的轨迹中恢复。

**突破**：将 Xavier 初始化的权重规模减半。
```
w1 = 0.5 × sqrt(2/(IN+H)) × randn(H, IN)
w2 = 0.5 × sqrt(2/(H+Out)) × randn(Out, H)
```
更小的初始权重 → 更小的初始 PID 增益 → 闭环从第一步就稳定 → 延时梯度能够正常学习。

## 核心技术要点

### 1. 代数环消除
`u(k)` 出现在被控对象方程右侧是不合理的，实际控制系统中控制量至少经过一步延迟才影响输出。

### 2. 完整延时梯度
当 `y(k)` 由 `u(k-1)` 驱动时，权重更新必须延迟一拍：
```
步 k:   存储 {e_pid, O2, dO3, dO2, I1}
步 k+1: 用存储状态 + e(k+1) 回传更新
```
近似条件：权重变化缓慢（小学习率），`∂u(k)/∂w(k) ≈ ∂u(k-1)/∂w(k-1)`。

### 3. 网络输入设计
`r(k) - r(k-1)` 替代 `e(k-1)` 作为网络输入，直接提供参考信号速度信息。对于跟踪时变信号至关重要。

### 4. 植物特化超参数
| 参数 | Plant1 | Plant2 | 理由 |
|---|---|---|---|
| Jacobian cap | 1.0 | **0.3** | 二阶慢动态需保守梯度 |
| Kd | 有 | **0** | 微分放大延迟效应 |
| Xavier 规模 | 1.0× | **0.5×** | 二阶对初始增益极度敏感 |
| 预训练 | 15ep diverse | **7ep ramp+step** | diverse 对二阶有害 |
| du_max | 1.0 | 0.5 | 二阶需保守控制增量 |

### 5. 固定 PID 整定：从单目标到多目标多信号

#### 5.1 原始整定的缺陷
初始 `pid_tune.m` 仅使用 ITAE = Σ k·|e(k)| 作为代价函数，且仅对阶跃信号（r=1）整定。这导致三个问题：
1. **不惩罚超调**：ITAE 只关心误差绝对值，超调不被显式惩罚
2. **仅阶跃整定**：参数对斜坡、正弦等时变信号泛化差（固定 PID 斜坡峰值误差曾达 1.14）
3. **评价体系不全**：MAE 无法反映超调量、调节时间、上升时间等动态品质

#### 5.2 多目标多信号整定
**代价函数**：`cost = Σ(ITAE/1000 + w_ov·σ² + w_rt·tr)` 对阶跃+斜坡+正弦三个信号求和
- σ：超调量（max(0, max(y)-r)/r）
- tr：上升时间（10%→90%）
- w_ov=2000, w_rt=0.5（超调惩罚为主）

**动态评价指标**：
| 指标 | 含义 |
|---|---|
| 超调量 σ% | max(0, max(y)-r)/r × 100 |
| 调节时间 ts | 误差进入 ±5% 且不再超出的步数 |
| 上升时间 tr | 输出从 10% 上升到 90% 的步数 |
| 稳态误差 ess | 最后 200 步平均绝对误差 |

#### 5.3 Z-N 法的失败
对 Plant2 线性二阶系统尝试 Ziegler-Nichols 阶跃响应切线法：
- 辨识结果：K=1.50, L=1.0, T=14.0
- Z-N PID：Kp=11.2, Ki=5.6, Kd=5.6
- 阶跃超调 164%，调节时间 2000（永不收敛）

**根因**：Z-N 法假设连续时间系统，不适用于 `u(k-1)` 单拍延迟的离散系统。离散延迟相当于额外的纯滞后环节，使 Z-N 参数过度激进。

#### 5.4 BP 兼容约束整定
无约束 fminsearch 找到的 PID 增益（Plant2: Kp=6.46, Kd=7.84）远超 BP 网络输出范围，导致 BP/Fix 比较不公平。解决方案：在 BP 增益范围内约束整定。
- Plant1: Kp_max=2, Ki_max=2, Kd_max=1
- Plant2: Kp_max=2, Ki_max=0.5, Kd_max=0（微分对 `u(k-1)` 延迟有害）

#### 5.5 整定结果对比

| | Plant1 旧 | Plant1 新 | Plant2 旧 | Plant2 新 |
|---|---|---|---|---|
| Kp | 0.15 | **0.46** | 1.38 | **1.94** |
| Ki | 0.63 | **0.34** | 0.16 | **0.21** |
| Kd | 0.29 | **≈0** | 0 | **0** |
| 阶跃超调 | 58% | **10%** | 11.8% | **0%** |
| 斜坡MAE | 0.048 | **0.023** | 0.002 | **0.001** |
| 正弦高频MAE | 0.025 | 0.064 | 0.212 | **0.141** |

整定逻辑修正后，固定 PID 的动态品质全面改善。Plant1 阶跃超调从 58% 降至 10%，Plant2 阶跃超调从 11.8% 降至 0%。
